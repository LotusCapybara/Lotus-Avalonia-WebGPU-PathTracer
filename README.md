# What is LotusRenderer
A real-time GPU path tracer built from scratch in C# using WebGPU compute shaders (WGSL).
All rendering runs entirely on the GPU via compute dispatches,no rasterization pipeline is used.

## Features

### Path Tracing
- Megakernel path tracer 
- Next Event Estimation (NEE) for Emissive Meshes
- PBR metallic/roughness workflow
- Dielectric transmission with Fresnel (Schlick approximation)
- Subsurface scattering via random walk (transmissive and opaque/shallow)
- 5 texture map slots per material (base color, roughness, metallic, normal, emission)
- Volumetric fog with HG phase function

### Acceleration Structure
- Two-level BVH (BLAS per mesh, TLAS over instances)
- SAH binned construction with adaptive bin counts
- glTF/GLB model loading via SharpGLTF

### Environment and Atmosphere
- HDR environment lighting (EXR) with orientation control
- Henyey-Greenstein homogeneous volumetric fog with distance-based extinction

### Post-Processing
- Exposure control
- Bloom (threshold, downsample, blur, composite)
- ACES filmic tone mapping
- Gamma correction
- Final RGBA8 packing for display

### Application
- Real-time viewport with progressive refinement
- Camera orbit (right-click drag) and WASD fly-through navigation
- Side panel UI for scene, camera, material, and environment controls
- Configuration persistence (JSON)
- Debug View Mode with different "buffer" views.

## Building and Running

### Requirements
- .NET 8.0 SDK
- Windows (the project builds as a WinExe). 
  - Mac: I tested a functional build on Mac, in a branch, still haven't merged it.
- A WebGPU-compatible GPU

### Build
```bash
dotnet build LotusRenderer/LotusRenderer.csproj -c Release
```

### Run
```bash
dotnet run --project LotusRenderer/LotusRenderer.csproj
```

### Assets

The `Data/` directory is not included in the repository. To use the renderer, create the following structure in the working directory:

```
Data/
  GLB/       # Place .glb model files here
  HDRi/      # Place .exr environment maps here
```

HDR environment maps can be found at [Poly Haven](https://polyhaven.com/hdris).

## Known Limitations

- BVH construction has issues with some complex multi-submesh GLB models
- Transmission BSDF is an approximation (no full refractive BTDF yet)
- No concrete lights, so no NEE for them either.
- Anisotropic roughness is not yet implemented (field exists in the material struct)
- No denoiser
